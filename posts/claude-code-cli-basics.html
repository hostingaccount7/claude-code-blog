This post explains a simple, beginner-friendly way to use Claude Code from the terminal (CLI). The exact commands can vary by version and operating system, so treat the examples as a pattern: learn how to discover what your install supports, then build a safe workflow.

1) Install and verify

After you install Claude Code, verify the CLI is available and see what it can do:

claude --help
claude --version

If your command is different (for example, installed via another tool), use the name that works on your system. The goal is to learn the habit: always check --help first.

2) Start inside a project folder

Create a folder for a small practice project. Keeping each project separate makes it easier to share and safer to review. 

mkdir my-project
cd my-project

3) A safe “first session” pattern

When you start, tell the tool what you want, your constraints, and how to verify results. Example request (adapt it to your CLI flow): 

Goal: build a simple static blog site (HTML/CSS/JS).
Constraints:
- keep it offline-friendly
- no paid services
- explain each change briefly
Verification:
- pages load without errors
- links work

4) Always ask for a plan before big changes

A good habit: ask for a short plan first, then do work step-by-step. This prevents surprise changes and helps you learn faster.

5) “Show me the diff” mindset

If you’re using Claude Code to modify files, get comfortable checking what changed. In any workflow (Git or not), the key skill is: review changes before you trust them.

  • Read the updated file(s) after changes
  • Run the project (or a small test) to confirm
  • Keep changes small and reversible

6) Common beginner mistakes

  • Too much at once: ask for small, focused edits instead of “build everything”.
  • Unclear success: always define “done” (what should work after the change).
  • Copy without understanding: ask the tool to explain one section at a time.
