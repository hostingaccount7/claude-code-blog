Claude Code has a set of built-in slash commands and keyboard shortcuts that most beginners never discover. Knowing them makes your sessions faster, cleaner, and easier to control. This is a reference you can come back to any time.

Starting and stopping

claude

Starts an interactive Claude Code session in your current folder. Claude Code can read and edit any file in that directory.

cd my-project
claude

claude "your prompt here"

Runs a single prompt non-interactively and exits. Useful for quick one-off tasks from a script or terminal without entering an interactive session. 

claude "explain what app.js does in one paragraph"

exit or Ctrl+C

Ends the current session. Your files are unchanged; only the conversation ends.

Conversation control

/clear

Wipes the entire conversation history in the current session without touching your files. Use it when a back-and-forth has gone off-track and you want to refocus with a clean prompt. 

/clear

/compact

Summarises the conversation so far into a shorter form to free up context space. Unlike /clear, it preserves a compressed memory of what was discussed. Use it in long sessions before context runs out. 

/compact

Ctrl+C during a response

Interrupts Claude Code mid-response. If a response is going in the wrong direction or is taking too long, press this to stop it and rephrase your prompt.

File and code operations

/add <file>

Explicitly adds a file to Claude Code's context so it can read and edit it. Useful when you want to be precise about which files are in scope. 

/add src/auth.py

Referring to files in prompts

You don't always need /add. Mentioning a filename in your prompt often causes Claude Code to read it automatically. Be explicit when the file is large or in a nested folder: 

Look at src/utils/format.py and explain the formatDate function.

Information and help

/help

Shows all available commands with a short description. Run this any time you forget a command name.

/help

/status

Shows current session information: how much context has been used, which model is running, and other session details. Check this in a long session to see how much context budget remains. 

/status

claude --version

Prints the installed version. Run this outside a session to confirm Claude Code is installed correctly.

claude --version

claude --help

Lists all command-line flags and options. Useful for flags like --model or --print.

Model selection

claude --model <model-id>

Starts a session with a specific Claude model. Useful if you want to use a faster or more powerful model than the default for a particular task. 

claude --model claude-haiku-4-5-20251001

Memory and project configuration

CLAUDE.md

Not a command, but the most important file to know about. Place a file called CLAUDE.md in your project root. Claude Code reads it automatically at the start of every session. Use it to set project conventions, technology choices, and rules so you don't repeat them every time. 

# CLAUDE.md example
Language: Python 3.11
Style: snake_case, 88-char line limit
Don't modify tests unless I ask.
Always explain changes before making them.

See the context tokens guide for more on how CLAUDE.md saves you token budget.

Keyboard shortcuts at a glance

  • ↑ / ↓ — scroll through previous prompts in the current session.
  • Ctrl+C — interrupt a running response.
  • Ctrl+L — clear the terminal display (does not clear conversation history; use /clear for that).
  • Tab — autocomplete file paths when typing them in prompts.

Full command quick-reference

  • claude — start an interactive session
  • claude "prompt" — run one prompt and exit
  • claude --version — check installed version
  • claude --help — list all CLI flags
  • /help — list all in-session commands
  • /clear — wipe conversation history
  • /compact — summarise history to save context
  • /add <file> — add a file to context
  • /status — show session and context info
