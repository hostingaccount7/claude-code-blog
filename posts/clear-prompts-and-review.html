Getting good results from Claude Code often comes down to two habits: writing clear prompts and reviewing every change. This post gives simple rules you can follow every time.

Write clear prompts

Be specific about the outcome

Instead of “fix my code,” say what should happen after the fix:

When I click Submit, the form should show “Thank you” and clear the fields.
Currently it does nothing. Fix only the submit handler.

Give context in one line

One sentence helps a lot: language, framework, or where the problem is.

This is a React form. The state is in App.js. The button is in Form.js.

Ask for one thing at a time

One request per message keeps changes small and easy to check. Break big tasks into steps.

Review every change

  • Read the diff: Skim what was added, removed, or changed before you accept.
  • Run the app: After each change, run or refresh and test the behavior you asked for.
  • Revert if unsure: If something looks wrong, undo and ask again with a clearer prompt.

Quick checklist

  1. Did I say what “done” looks like?
  2. Did I give one small task, or many?
  3. Did I read the changes before accepting?
  4. Did I run the code to verify?
