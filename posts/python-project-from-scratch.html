What you'll build

A command-line quiz program: you give it questions and correct answers, it quizzes you, and it tells you your score at the end. The complete program is about 40 lines of Python—simple enough to understand every line, real enough to be genuinely useful.

You will use Claude Code as a collaborator, not as someone who types for you. That means planning first, prompting one step at a time, and reading everything before you accept it.

Before you start: check your tools

# Confirm Python is installed
python --version   # or python3 --version

# Confirm Claude Code is running
claude --version

If either command fails, see the CLI basics guide for setup steps.

Step 1: Create the project folder

mkdir quiz-app
cd quiz-app
git init

Start git immediately. You will commit after each working step so you can always go back.

Step 2: Plan before prompting

Write your plan in a file called plan.txt before opening Claude Code. It can be rough: 

Quiz program plan
-----------------
- Store questions as a list of dictionaries: {question, answer}
- Loop through questions and ask user for input
- Compare input to answer (case-insensitive)
- Count correct answers
- Print final score at the end

Having a written plan means your prompts will be specific. Specific prompts produce smaller, easier-to-review suggestions.

Step 3: Start Claude Code and open your project

claude

Claude Code will start in your current folder (quiz-app).

Step 4: Prompt one piece at a time

Do not paste your whole plan and ask for the finished program. Instead:

First prompt

Create a file called quiz.py. Add a Python list called QUESTIONS. Each item should be a dictionary with two keys: "question" and "answer". Add three example questions about basic Python syntax. Don't add anything else yet.

Read the suggestion. Does it look like a list of dictionaries? Are the example questions sensible? If yes, accept it and commit: 

git add quiz.py
git commit -m "Add questions list"

Second prompt

Add a function called run_quiz that loops through QUESTIONS, prints each question, reads the user's answer with input(), and compares it to the correct answer. The comparison should ignore uppercase and lowercase. Count correct answers in a variable called score. Don't print the score yet.

Read the function. Trace it by hand: what happens on the first question? The second? Does score start at zero? Commit if it's correct.

Third prompt

After the loop ends, print the user's score out of the total number of questions. Then add an if __name__ == "__main__": block that calls run_quiz(). Don't change anything else.

Step 5: Run it and test by hand

python quiz.py

Answer the questions deliberately: get one right on purpose, one wrong on purpose. Check that the final score matches what you expected. This is your manual test—always do it before committing a feature.

Step 6: Extend it yourself

Now that the core works, try adding features with one prompt each. Do them one at a time:

  • Shuffle the question order using Python's random module.
  • Add a time limit: if the user takes more than 10 seconds, mark the question wrong.
  • Load questions from a text file instead of the hard-coded list.
  • Keep a high-score record in a file so the user can try to beat their own record.

Each of these is one prompt and one commit. If a feature breaks the program, you have a clean commit to revert to.

What you learned by doing this

  • Planning before prompting produces smaller, clearer changes.
  • Committing after each working step means mistakes are always reversible.
  • Testing by hand—not just running without errors—catches logic mistakes that Claude Code cannot see.
  • Extending one feature at a time keeps the code understandable.

These habits work for 40-line programs and for 40,000-line programs. The scale changes; the habits do not.

The finished program (for reference)

import random

QUESTIONS = [
    {"question": "What keyword defines a function in Python?", "answer": "def"},
    {"question": "What is the output of len('hello')?", "answer": "5"},
    {"question": "Which method adds an item to a list?", "answer": "append"},
]

def run_quiz():
    questions = QUESTIONS[:]
    random.shuffle(questions)
    score = 0

    for item in questions:
        user = input(item["question"] + " ").strip().lower()
        correct = item["answer"].strip().lower()
        if user == correct:
            print("Correct!")
            score += 1
        else:
            print(f"Wrong. The answer was: {item['answer']}")

    print(f"\nYour score: {score}/{len(questions)}")

if __name__ == "__main__":
    run_quiz()

Treat this as a reference, not a shortcut. The goal is to build it step by step so you understand every line.
