Understanding /insights: Read Your Session Stats and Work Smarter

Claude Code gives you a built-in dashboard for your current session. The command is called /insights, and it shows you exactly how much of your context window you have left, how many turns you have taken, and which model is doing the work. Once you know how to read it, you can make smart decisions before you run into problems.

How to Run It

Type this at any point during your session:

/insights

You will see output that looks something like this:

Session Insights
────────────────────────────────
Tokens used:        42,318
Context window:     33%
Turns:              14
Model:              claude-sonnet-4-5
────────────────────────────────

If your version of Claude Code does not show /insights, try /status instead — it covers similar ground.

How to Read the Output

Tokens used

This is the running total of tokens consumed in your session — your messages, Claude's replies, and any file contents that were read. Think of it as a fuel gauge for the conversation so far.

Context window percentage

This tells you how full the context window is. The context window is the maximum amount of text Claude can hold in memory at once. When it fills up, older parts of the conversation get dropped, and Claude may lose track of earlier work.

Turns

One turn is one message from you plus one reply from Claude. This number helps you gauge how long and complex the session has become.

Model

Shows which Claude model is handling your session. Useful to confirm you are on the model you expect.

Three Ways to Act on What You See

  1. Context percentage above 60%? Run /compact now. Do not wait until it hits 90%. Running /compact early summarises the conversation and frees up space before anything important gets dropped.
  2. High turn count but little progress? Start a fresh session. Long, tangled sessions often slow Claude down. If you have taken 30+ turns and feel like you are going in circles, close and reopen with a cleaner, focused prompt.
  3. Tokens spiking faster than expected? You are reading too many large files. Ask Claude to look at specific functions or line ranges instead of entire files. Every file read costs tokens, so being precise goes a long way.

/insights vs /status

/insights is focused on session analytics — it is the command to reach for when you want numbers about your usage. /status tends to show broader information about the current task state, active files, and what Claude is working on. Use /insights to monitor your token budget, and /status to check what Claude thinks it is doing right now.

Quick-Reference Table

Field What it means Action if it looks high
Tokens used Total tokens consumed this session Be more targeted with file reads
Context window % How full the context window is Run /compact above 60%
Turns Number of back-and-forth exchanges Consider a fresh session above 30
Model Which Claude model is active No action needed — just for awareness

Build the Habit: Check Every 10 Turns

Every 10 turns, pause and type /insights. It takes two seconds and tells you whether you are still in good shape or whether it is time to compact or reset. Catching a full context window early is far easier than recovering from one that has already overflowed.
