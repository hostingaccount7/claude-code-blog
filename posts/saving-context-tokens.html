Every Claude Code session has a context window—a limit on how much text the model can hold in memory at once. When you hit that limit, older parts of the conversation start getting dropped, responses can become less accurate, and long sessions slow down. Learning to use context efficiently is one of the highest-leverage habits you can build.

What uses context tokens?

Everything in a session counts: your prompts, Claude's responses, every file that gets read, every diff that gets shown, and the conversation history from the start of the session. Large files read in full, vague back-and-forth, and repeated explanations are the biggest drains.

1. Start fresh sessions for new tasks

A session that started with "fix the login bug" should not also be used to "add a new dashboard page." Each task gets its own session. When a task is done, close the session and open a new one. You keep the full context budget for each new task rather than starting with it half-spent. 

# End the current session
exit

# Start a fresh one for the next task
claude

2. Use /clear to reset mid-session

If you are mid-session and the conversation has drifted into a long back-and-forth that is no longer relevant, run /clear. This wipes the conversation history while keeping your files unchanged. You can then re-focus with a single tight prompt. 

/clear

Use it when: you've been debugging and the conversation is full of failed attempts, or you asked several exploratory questions and now want to switch to actually writing code.

3. Give Claude only the files it needs

Avoid prompts like "look at all the files in this project." Instead, name the specific file:

Read only auth.py. Don't open any other files unless I ask.

Every file Claude reads consumes tokens. If you have a 2,000-line file but the bug is on line 47, say so:

In utils.py, look only at the calculate_total function around line 47.

4. Keep prompts short and specific

Long preambles waste tokens. Compare:

  • Wasteful: "Hi, I've been working on this project for a few weeks and I'm having some trouble with a function I wrote last Tuesday that is supposed to calculate the total price including tax…"
  • Efficient: "In cart.py, the calculate_total function returns the wrong value when tax is 0. Fix it."

Background you already explained in this session doesn't need to be repeated. If you are starting fresh, one sentence of context is usually enough.

5. Ask for compact responses when you don't need explanation

By default Claude Code explains its reasoning. When you just want the output, say so:

Rename all instances of getUserData to fetch_user_data in api.js. Show me only the diff, no explanation.

Shorter responses mean more room in the context window for the actual work.

6. Use CLAUDE.md to avoid re-explaining your project

If you find yourself typing the same background in every session ("this is a Flask app, we use PostgreSQL, don't use f-strings"), put it in a CLAUDE.md file at the root of your project. Claude Code reads this file automatically at the start of every session. One short, well-written file saves you dozens of repeated prompt tokens across many sessions. 

# CLAUDE.md
Language: Python 3.11
Framework: Flask
Database: PostgreSQL via psycopg2
Style: snake_case, no f-strings, max line length 88

7. Prefer /compact over letting context fill up silently

When a session is getting long and you want to continue rather than start fresh, run: 

/compact

This asks Claude Code to summarise the conversation so far into a shorter form, freeing up space while preserving the key facts. Use it when you notice responses starting to feel less precise or when the session has been running for a long time.

Signs your context is running low

  • Claude Code references something incorrectly that you discussed earlier in the session.
  • Responses start repeating suggestions you already rejected.
  • The model seems to forget constraints you set at the beginning ("I told you not to modify that file").

When you notice these signs, run /compact or start a new session and give one sentence of fresh context.

Quick reference

  • New task → new session. Don't share context budgets between unrelated tasks.
  • /clear — wipe conversation history, keep files.
  • /compact — summarise history to free up space.
  • Name specific files instead of asking Claude to explore the whole project.
  • CLAUDE.md — write project context once, not every session.
  • Short prompts, short responses — ask for diffs, not explanations, when you just want the code.
